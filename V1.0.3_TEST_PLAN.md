# DSI Redline v1.0.3 — UI Test Plan

Targeted checks for the **bug fixes and new features** in v1.0.3, plus a
**regression** pass on the areas those changes touched. Tick each box; note
`❌ <detail>` on any failure.

> [!note] Prepare
> - [ ] A **vector** AutoCAD-Electrical PDF (real text layer, wire numbers + device tags, a title block).
> - [ ] A **scanned / image-only** PDF.
> - [ ] A previously **saved `.marked.pdf`** (make one: open a PDF, add a mark, Save markup).
> - [ ] *(For Bug 4)* a drawing whose sheet number is the **lesser of two numbers in the bottom-right corner**.

## A. Bug fixes

### A1 — Rotate grip vs. text selection
- [ ] Select tool active. Click a rectangle/highlight to select it → resize/rotate grips appear.
- [ ] **Drag the rotate grip (above the mark)** → the mark **rotates** (previously it started a text selection).
- [ ] Drag a corner grip → the mark **resizes** (still works).
- [ ] Drag the body → the mark **moves** (still works).
- [ ] Place a mark **right next to selectable text**; grab its grip → it resizes/rotates (does **not** select text).
- [ ] Click/drag on **empty page area** → text selection still works (not broken).

### A2 — Ctrl+F re-search of stale text
- [ ] Find something with `Ctrl+F`; close the bar with `Esc`.
- [ ] Press `Ctrl+F` again → the previous text is still there **and matches re-highlight immediately**.
- [ ] With stale text showing no highlights, press **Enter** → it searches and lands on the first match.

### A3 — Delete key confirms
- [ ] Select a comment / text box / highlight / pen mark, press **`Delete`** → an "are you sure?" prompt appears (matches right-click / 🗑 behavior).
- [ ] Confirm → it deletes (undoable with `Ctrl+Z`). Cancel → nothing happens.
- [ ] Select **several** marks, press `Delete` → **one** prompt covering all of them.

### A4 — Sheet auto-fill from the corner *(best-effort — verify on YOUR drawings)*
- [ ] Open a vector drawing; add a TODO on a page whose sheet number is in the bottom-right corner.
- [ ] The TODO row's **Sheet** column shows that page's number (the lesser of the two corner numbers).
- [ ] A page where it can't tell → blank, and you can still type it in (editable).
> [!warning] This heuristic is unverified against your exact title-block layout. If sheets fill in **wrong** or stay **blank** on your real drawings, note the values seen and (ideally) share a sample page so the corner region can be tuned.

### A5 — TODO rows don't drag
- [ ] Try to **drag a TODO row** onto another → nothing happens (no reorder, no nesting).

### A6 — Wire / Component double-click jumps to the drawing
- [ ] Extract wire numbers; **double-click a row** → the Viewer opens and scrolls to that wire's **location on the drawing**, with a brief orange pulse.
- [ ] Double-click a label that appears in several places → it jumps to the **first** occurrence (no error).
- [ ] Same check on the **Component Labels** tab.

### A7 — New family codes
- [ ] In **Settings ▸ Component labels**, confirm the family list includes `CBL, DV, EN, DN, GND, PDB, PRS, PW, SCR, SE, X`.
- [ ] Extract on a drawing using one of these (e.g. `GND-…`) → it is **not** flagged "unknown family".

### A8 — Family edits refresh flags without re-extract
- [ ] Extract component labels; find a tag flagged **`unknown family`**.
- [ ] Open **Settings**, add that family code, click **OK**.
- [ ] The Component Labels table updates **immediately** — the flag clears, no re-extract needed.
- [ ] Your include (✓) ticks on the rows are preserved.

## B. New features

### B1 — Block opening an already-open PDF
- [ ] With a PDF open, **File ▸ Open** the **same file** → a popup says it's already open; the document is **not** reloaded.
- [ ] Opening its `*.marked.pdf` while the original is open is also treated as the same document.
- [ ] Opening a **different** PDF still works normally.

### B3 — Export hotkey
- [ ] With a document open, press **`Ctrl+Shift+E`** → the "Export annotated PDF…" save dialog opens; it writes the annotated copy.
- [ ] **File** menu shows the `Ctrl+Shift+E` shortcut next to "Export annotated PDF…".

### B4 — One markup DB / one `.marked.pdf`
- [ ] Open `drawing.pdf`, add a mark, **Save** → creates `drawing.marked.pdf` + `drawing.markup.db`.
- [ ] **Open `drawing.marked.pdf`** → your marks are there; **no second `.markup.db`** appears.
- [ ] Add another mark and Save → it updates **`drawing.marked.pdf`** (there is **no** `drawing.marked.marked.pdf`), and existing marks are **not duplicated**.
- [ ] Move a `.marked.pdf` to a folder without its `.markup.db`, open it → a notice says a new markup database was started.

### B5 — TODO filter across all columns
- [ ] In the TODO **Filter** box, type a **tag** → only items with that tag show.
- [ ] Type a **commenter** name, a **sheet** number, or a **page** number → matching rows show.
- [ ] Clear the box → all rows return.

## C. Regression checks (areas these changes touched)

> [!note] The fixes touched the viewer's mouse handling, search, save/open, the TODO panel, and the component classifier — re-verify the neighbours.

### Viewer / marks
- [ ] **Text selection + `Ctrl+C`** on empty page area still selects and copies.
- [ ] Create each mark type (highlight, pen, comment, text box, rectangle, arrow) — all still draw.
- [ ] Move / resize / rotate a mark; **resize a rotated mark** keeps the opposite corner anchored.
- [ ] **Undo / redo** across create / move / resize / delete.
- [ ] Delete via **right-click ▸ Delete** and via the **Comments 🗑** still confirm and work.
- [ ] **In-memory rotate (↺/↻)** still rotates the view with marks; region-pick / crop still lands correctly while rotated.
- [ ] **Find**: typing a fresh query still searches live; `Enter`/`Shift+Enter` step matches; count `i/n` correct.

### Panels & navigation
- [ ] **Comments sidebar**: click-to-flash, filters, sort, delete (3 ways) still work.
- [ ] **TODO**: grouping (Page / Sheet / Commenter / None), header sort, **edit Text/Sheet/Tag**, Pg double-click jump, right-click "Go to in PDF" still work.
- [ ] **Reveal in TODO / Comments** from a right-clicked mark still works.
- [ ] **Navigation** thumbnails / bookmarks still jump (and switch to the Viewer from another tab).
- [ ] **Wire / Component extract** (AI/OCR dropdown, progress, cancel), table sort/filter, and **export** still work.

### Storage round-trips
- [ ] Open original → mark → Save → close → **reopen original** → marks, TODO state, tags, sheet numbers, wire/component cache all persist.
- [ ] Open a **colleague-annotated** PDF → external annotations still import with authors; Save → they aren't duplicated.
- [ ] **PDF Tools** (extract/split/delete/rotate/combine/insert/swap/convert/crop) still operate and write new files.
